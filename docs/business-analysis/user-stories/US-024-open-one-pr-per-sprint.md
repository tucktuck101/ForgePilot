## Summary

Capture the Codex need to keep each sprint reviewable and traceable in one draft PR.

## Story

As Codex, I want to open one draft PR per scoped sprint so that work is reviewable and traceable.

## Acceptance Criteria

- A branch is created for the sprint.
- The PR links to its issue.
- The PR contains summary, tests, risks, and approval gates.
- The PR remains unmerged until approval.
