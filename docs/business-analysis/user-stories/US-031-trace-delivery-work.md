## Summary

Capture the Technical Reviewer need to trace delivery changes and decisions.

## Story

As a Technical Reviewer, I want Issues, PRs, sprint notes, and docs linked so that I can trace what changed and why.

## Acceptance Criteria

- Issues link to docs.
- PRs link to issues.
- Sprint notes link to PRs.
- Test reports link to sprint notes.
- ADRs link to relevant issues/PRs/docs.
