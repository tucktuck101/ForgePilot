## Summary

Capture the Codex need to validate sprint work before treating it as complete.

## Story

As Codex, I want to run relevant validation before marking a sprint complete so that code is not treated as done just because it was written.

## Acceptance Criteria

- Relevant checks are run.
- Failures caused by the sprint are fixed.
- Unrelated failures are documented.
- Test report is updated.
- Sprint Definition of Done is satisfied.
