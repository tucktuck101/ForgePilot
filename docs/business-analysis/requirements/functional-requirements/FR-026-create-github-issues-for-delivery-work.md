## Summary

ForgePilot shall use GitHub Issues as the primary vehicle for executable delivery work. Each task in the backlog should be represented by a GitHub Issue with appropriate metadata.

## Details

- After scope and design are approved, Codex creates a backlog of issues based on the user stories, technical tasks, and other work items.
- Each issue includes a title, description, acceptance criteria, links to relevant documents, risk level, dependencies, and definition of done.
- Issues are assigned labels, milestones, or assignees as appropriate.
- Issues form the basis for planning sprints and tracking progress.

## Acceptance Criteria

- All delivery tasks are tracked as GitHub Issues.
- Issues follow the structure defined in FR-029 (issue structure).
- Issues link back to the relevant requirements, stories, or design documents.
- Codex does not rely solely on internal notes to manage tasks; tasks are public in GitHub Issues.

## Notes

GitHub Issues integrate with the PR workflow and provide transparency for team members. Using issues ensures that work is discoverable and auditable.
