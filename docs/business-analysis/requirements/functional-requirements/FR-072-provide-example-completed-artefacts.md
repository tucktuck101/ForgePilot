## Summary

Include example completed artefacts to demonstrate how finished documents should look.

## Details

In addition to blank templates, ForgePilot should provide example completed artefacts illustrating the expected level of detail and formatting. These examples serve as references for both humans and Codex when creating new artefacts. Examples should be placed in a dedicated folder (e.g., `examples/completed-artefacts/`) and clearly marked as examples so they are not mistaken for the project’s live documents.

## Acceptance Criteria

- A separate directory contains example completed artefacts for key document types (such as a filled-out product brief, functional requirement, persona, and user story).
- Example files are clearly labelled as examples and not included in the live documentation indices.
- AGENTS.md or relevant skills warn Codex not to treat examples as project truth.

## Notes

Examples help set expectations for documentation quality and make the template more approachable for new users.
