## Summary

ForgePilot shall produce polished, structured documentation.

## Details

All project artefacts created by ForgePilot should be written in Markdown using clear and consistent headings, doc‑object metadata tables, and concise prose. Documents should be organised in logical sections with an appropriate level of detail. The style should be professional and easy to read, serving both human users and Codex. Documentation should include update rules to indicate when and why a file should be modified. Avoid embedding long sentences in tables; tables should only be used for keywords, phrases, numbers, or images.

## Notes

High-quality documentation improves comprehension, handover, and future maintenance.
