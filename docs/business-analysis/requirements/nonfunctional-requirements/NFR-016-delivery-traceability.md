## Summary

Delivery work must be traceable.

## Details

ForgePilot should ensure that all delivery work is traceable through GitHub Issues, pull requests, sprint notes, test reports, release notes, and documentation. Issues should link to relevant documents and acceptance criteria; PRs should link to their issues; sprint notes should summarise what was done and any decisions made; test reports should link to related sprints; and release notes should summarise changes across sprints. This traceability allows anyone to follow the evolution of the project from idea to prototype.

## Notes

Clear traceability streamlines audits, handovers, and future feature planning.
